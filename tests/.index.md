# tests — PHX pytest suite

Pytest suite for the PHX library. Directory layout mirrors the source: `test_model/` covers the in-memory PHX model classes; the `test_from_*` / `test_to_*` folders cover importers and exporters (HBJSON, WUFI XML, PHPP, PPP, METr JSON). Run with `python -m pytest tests/` from the repo root.

## Contents

### `test_model/` (59 py)
Unit tests for the in-memory PHX model classes, one subfolder per model area: `test_building/`, `test_certification/`, `test_components/`, `test_constructions/`, `test_elec_equip/`, `test_enums/`, `test_geometry/`, `test_hvac/`, `test_loads/`, `test_project/`, `test_schedules/`, `test_spaces/`, `test_util_patterns/`. Mirrors `PHX/model/`.

### `test_from_HBJSON/` (15 py)
Honeybee (HBJSON) → PHX importer tests. Subfolders: `test_read_HBJSON_file/`, `test_create_project/`, `test_create_building/`, `test_create_assemblies/`, `test_create_hvac/`. Mirrors `PHX/from_HBJSON/`.

### `test_from_WUFI/` (32 py)
WUFI-Passive XML → PHX importer tests (pydantic schema parsing). Subfolders by domain: `test_project/`, `test_variants/`, `test_site/`, `test_geometry/`, `test_envelope_types/`, `test_hvac/`, `test_home_devices/`, `test_patterns/`. Mirrors `PHX/from_WUFI_XML/`.

### `test_to_WUFI_xml/` (81 py — largest)
PHX → WUFI-Passive XML exporter tests. Subfolders: `test_project/`, `test_building/`, `test_geometry/`, `test_construction/`, `test_certification/`, `test_climate/`, `test_ground/`, `test_hvac/`, `test_elec_equip/`, `test_schedule/`, `test_bug_fixes/`, and `test_reference_cases/` (end-to-end HBJSON-in → XML-out comparison; see `to_xml_reference_cases` in conftest).

### `test_to_METr_JSON/` (3 py)
PHX → METr JSON exporter tests (METr = next-gen WUFI format). `test_reference_cases/test_metr_json_output.py` is the end-to-end output comparison. Newest exporter — sparse coverage relative to the others.

### `test_PHPP/` (14 py)
PHPP (Excel) interop tests. Subfolders: `test_phpp_model/`, `test_sheet_io/`. Mirrors `PHX/to_PHPP/` + `PHX/PHPP/`. Includes `test_xl_app.py` (XLConnection facade incl. worksheet-name/column-read caching) and `test_xl_data_merge_rows.py` (section block-write batching).

### `test_xl_replay/` (4 py)
Record/replay regression tests for the PHPP write path: `fake_xl_framework.py` (in-memory cell-state fake of the xlwings surface), `test_replay_invariant.py` (replays the full export against a recorded fixture and requires an exactly-identical final cell-state — 0.3 s, no Excel). Fixtures in `fixtures/` (recorded via `scripts/perf/record_replay_fixture.py`).

### `test_scripts_perf/` (3 py)
CI-safe unit tests for the `scripts/perf/` Tier-0 harnesses (H1 profiler, H2 read-back spec engine) against the `xl_typing` doubles. Includes round-trip-count regression guards (e.g. sheet-name reads must not scale with facade calls).

### `test_to_PPP/` (4 py)
PHX → PPP exporter tests. No reference data files.

### `reference_files/`
Expected-output fixtures for the end-to-end reference cases, organized by source pipeline (see Notes). Data files only — no Python.

### `_source_gh/`
`hbph_test_models.gh` — the Grasshopper definition used to author the `from_grasshopper_tests` HBJSON reference models. Binary; regenerate reference HBJSON from here.

## Key Files

- `conftest.py` — shared pytest fixtures (geometry primitives, model objects) and the `reference_files` path plumbing. Also home of `reset_class_counters` (resets `_count` ClassVars for stable IDs) and `to_xml_reference_cases` parametrization.
- `reference_files/from_WUFI/metr_json/CONTEXT.MD` — hand-written notes on the from-WUFI METr reference set.

## Notes

- **`live_excel` marker:** tests marked `@pytest.mark.live_excel` drive a real Excel application and are **deselected by default** (`addopts` in pyproject.toml). Run manually with `pytest -m live_excel`; never in CI.
- **`reference_files/` is organized by source pipeline, not by target:**
  - `from_grasshopper_tests/` — models authored in Grasshopper (`_source_gh/`); has `hbjson/`, `metr_json/`, `ppp/`, `wufi_xml/` subfolders. Includes real-project fixtures (`_la_mora`, `_ridgeway`, `_arverne_d_no_win`, `School`).
  - `from_hb_json_tests/` — expected outputs from the HBJSON→(XML|METr) pipeline; `metr_json/`, `ppp/`, `wufi_xml/`.
  - `from_WUFI/` — round-trip fixtures parsed from real WUFI XML; `metr_json/`, `wufi_xml/`.
- Canonical fixture models recurring across pipelines: `Default_Model_Single_Zone`, `Default_Room_Single_Zone`, `Multi_Room_Complete`.
- Regenerating reference files churns many of these at once (they show up together in `git status`) — expected when the exporters change.
- Every `test_*` subfolder carries an `__init__.py`; `__pycache__/` dirs are build noise (excluded here).
