# reference_files — expected-output fixtures for end-to-end tests

Golden input/output files for the PHX reference-case tests (the `to_xml_reference_cases` / METr / PPP parametrized end-to-end comparisons). **Organized by SOURCE pipeline, then by target format.** These files churn together in `git status` whenever an exporter changes — that's expected, not a bug. See parent `../.index.md` for how the test suite consumes them.

## Contents

### `from_grasshopper_tests/`
Models authored in Grasshopper (source def: `../_source_gh/hbph_test_models.gh`). Holds the HBJSON inputs plus their expected exports.
- `hbjson/` — input models (`.hbjson` + one `.json`).
- `wufi_xml/` — expected WUFI XML. Includes real-project fixtures (`_la_mora`, `_ridgeway`, `_arverne_d_no_win`, `School`) alongside the canonical `Default_Model_Single_Zone` / `Multi_Room_Complete`.
- `metr_json/` — expected METr JSON.
- `ppp/` — empty placeholder (no fixtures yet).

### `from_hb_json_tests/`
Expected outputs of the HBJSON → (XML | METr) pipeline for the two canonical models only (`Default_Model_Single_Zone`, `Multi_Room_Complete`).
- `wufi_xml/`, `metr_json/` — expected exports. `ppp/` — empty placeholder.

### `from_WUFI/`
Round-trip fixtures: real WUFI XML parsed back in.
- `wufi_xml/` — source XML (`_la_mora`, `_ridgeway`, `School`).
- `metr_json/` — METr JSON generated by WUFI-Passive from those XML sources (see CONTEXT.MD).

## Key Files

- `from_WUFI/metr_json/CONTEXT.MD` — notes that these METr files are generated by WUFI-Passive from source XML and serve as inputs for the new METr software. Original source XML referenced there lives outside this tree (`_test_reference_files_xml/`).

## Notes

- Canonical fixtures recurring across all pipelines: `Default_Model_Single_Zone`, `Default_Room_Single_Zone`, `Multi_Room_Complete`. Underscore-prefixed names (`_la_mora`, `_ridgeway`, `_arverne_d_no_win`) are real-project models.
- `ppp/` subfolders exist but are empty — placeholders for future PPP reference fixtures (`test_to_PPP/` currently has no reference data).
- Mixed extensions in `from_grasshopper_tests/hbjson/` (`.hbjson` vs `.json`) are both Honeybee JSON — the extension is inconsistent, not meaningful.
- To regenerate the HBJSON inputs, open `../_source_gh/hbph_test_models.gh` in Grasshopper.
