# PHX/model/ — the in-memory PHX object graph

The normalized model that `from_*` importers build and `to_*` exporters read. Authoritative object-graph reference: `../../docs/reference/phx-model-reference.md`.

## Key modules

- `project.py` — top-level project container.
- `building.py`, `components.py`, `geometry.py`, `shades.py` — building/envelope/geometry.
- `constructions.py`, `assembly_pathways.py`, `ground.py` — assemblies and ground/foundations.
- `spaces.py`, `certification.py`, `elec_equip.py`, `phx_site.py` — spaces, PH certification, equipment, site.
- `utilization_patterns.py` — utilization patterns.

## Subpackages

- `enums/` — enumerations (building, hvac, certification variants PHI/Phius, site, foundations).
- `hvac/` — HVAC devices/systems: `heating.py`, `heat_pumps.py`, `cooling_params.py`, `ducting.py`, `piping.py`, `renewable_devices.py`, `collection.py`.
- `loads/`, `programs/`, `schedules/` — the loads / programs / schedules triad (lighting, occupancy, ventilation in each).

## Notes
- Many classes carry a `_count` ClassVar for ID numbering — the `reset_class_counters` test fixture resets these.
- Tests: `../../tests/test_model/`.
