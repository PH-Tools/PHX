# PHX/ — the converter package

The whole library. Structure is `from_* → model → to_*` (see `../context/ARCHITECTURE.md`; authoritative deep docs in `../docs/dev/architecture.md`).

## Subpackages

- `model/` — the in-memory PHX object graph (building, components, constructions, geometry, hvac, loads, programs, schedules, enums). See `model/.index.md`.
- `from_HBJSON/` — build a PHX model from Honeybee-PH HBJSON (primary source). See `from_HBJSON/.index.md`.
- `from_WUFI_XML/` — build a PHX model from WUFI XML (pydantic schemas). See `from_WUFI_XML/.index.md`.
- `from_PHPP/` — build a PHX model from a PHPP file.
- `to_WUFI_XML/` — write WUFI XML. See `to_WUFI_XML/.index.md`.
- `to_PPP/`, `to_METr_JSON/` — write PPP / METr-JSON targets.
- `PHPP/`, `xl/` — PHPP Excel write path + Excel-interop (`xlwings`).

## Entry points (top level)

- `hbjson_to_wufi_xml.py`, `hbjson_to_phpp.py`, `hbjson_to_ppp.py`, `hbjson_to_metr_json.py` — end-to-end CLIs.
- `run.py` — **Python-2.7 Grasshopper shim** (excluded from formatting; keep Py2.7-safe).

## Notes
- Tests: `../tests/` (mirrors this layout). PHPP write path guarded by the `test_xl_replay` golden invariant.
