# planning/features/

One folder per new-capability plan, lowercase kebab-case: `planning/features/<slug>/`.

Folder contract (drop the optional files for small items):

```
<slug>/
  README.md   # router: scope, read order
  PRD.md      # what/why — behavior contract
  STATUS.md   # current state, next step, blockers
  PLAN.md     # optional implementation sequence
```

Register every active folder in `planning/STATUS.md`. When complete, fold the outcome into `context/` (or the authoritative `docs/` deep-dives) and move the folder to `planning/archive/`. See `planning/.instructions.md` for the full rules and status vocabulary.
